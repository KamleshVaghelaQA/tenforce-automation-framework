✔ Framework Choice Justification
Selenium for UI automation
Cucumber (BDD) for readable test scenarios
POM for maintainability and scalability
Hooks for setup/teardown
Extent Reports for rich reporting
Cross-browser fallback for reliability


✔ Architecture (Short)
This framework follows a layered architecture with separation of concerns:
Feature files define behavior (BDD)
Step Definitions handle logic
Page Objects encapsulate UI interactions
Hooks manage lifecycle
Utilities provide reusable services like driver and reporting
